# Python 容器操作與推導式/生成器深度解析

## 概述

本文深入探討 Python 中的容器操作、列表推導式、字典推導式以及生成器表達式的應用。通過分析這些語法結構的原理和效能，為開發者提供高效數據處理的技術洞見。

## 語法解析

### 列表推導式 (List Comprehensions)

列表推導式提供了一種簡潔的方式來創建新列表。其基本語法為：

```python
[expression for item in iterable if condition]
```

- `expression`：對每個元素執行的運算
- `item`：迭代變數
- `iterable`：可迭代對象
- `if condition`：可選的過濾條件

### 字典推導式 (Dictionary Comprehensions)

字典推導式用於創建新字典：

```python
{key_expression: value_expression for item in iterable if condition}
```

### 生成器表達式 (Generator Expressions)

生成器表達式類似於列表推導式，但返回生成器對象：

```python
(expression for item in iterable if condition)
```

生成器使用 `()` 而非 `[]`，並且是惰性求值的。

## 比較表格

| 特性 | 列表推導式 | 傳統循環 | 生成器表達式 |
|------|------------|----------|--------------|
| 內存使用 | 立即創建完整列表 | 依賴循環邏輯 | 惰性求值，節省內存 |
| 執行時機 | 立即執行 | 逐步執行 | 按需執行 |
| 返回類型 | list | 依賴代碼 | generator |
| 適用場景 | 小數據集處理 | 複雜邏輯 | 大數據集或無限序列 |
| 可讀性 | 高 | 中等 | 高 |

## Big O 複雜度分析

### 時間複雜度

- **列表推導式**：$O(n)$ - 線性時間，與輸入大小成正比
- **字典推導式**：$O(n)$ - 同上
- **生成器表達式**：$O(1)$ 初始化，$O(1)$ 每次迭代 - 惰性求值使初始化為常數時間

### 空間複雜度

- **列表推導式**：$O(n)$ - 存儲完整結果列表
- **字典推導式**：$O(n)$ - 存儲完整結果字典
- **生成器表達式**：$O(1)$ - 不預先存儲數據，只保存當前狀態

## 實用範例

### 基本列表推導式

```python
nums = [1, -2, 3, -4]
positives = [n for n in nums if n > 0]  # [1, 3]
```

### 字典推導式

```python
pairs = [('a', 1), ('b', 2)]
lookup = {k: v for k, v in pairs}  # {'a': 1, 'b': 2}
```

### 生成器表達式應用

```python
nums = [1, -2, 3, -4]
squares_sum = sum(n * n for n in nums)  # 計算平方和，節省內存
```

## 效能考量

- 對於小數據集，列表推導式通常是最簡潔的選擇
- 處理大數據時，生成器表達式可以顯著減少內存使用
- 推導式在簡單轉換和過濾任務中比傳統循環更高效

## 結論

掌握推導式和生成器表達式能夠讓 Python 代碼更加簡潔和高效。通過理解它們的語法、比較不同方法的特性以及複雜度分析，開發者可以根據具體場景選擇最適合的數據處理方式。
